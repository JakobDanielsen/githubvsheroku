Github vs Heroku

How to GitHub How to Heroku

What are the differences?

The main differences between hosting a website on github versus heroku is that github only allows static contents wich is easier to make but makes all code public and very transparent instead of using responses and requests. In other words you can not host a page on github that uses express.js or other similar frameworks. Heroku lets you host more freely and its possible to use websites with requests and responses like node express.js.

This site is hosted on github and can therefore not use certain npm packets such as express.js or EJS

While chosing either Heroku or GitHub, they both work because of a program called "git".

How to host on Github

For your static website

  1. First of all head over to you project page and go to settings. We're going to change the project visibility.
  2. Then you scroll down to the project visibility. You can skip this step if you know your project is already public. The project needs to pe public in order to be hosted on Github.
  3. Make it public by checking off here.
  4. Then go to the pages section.
  5. The last step is to click on select branch and switching from None to main, or the name of the branch you want to upload.

    6. Now your site will be up in a minute or two and will also be updated every time you push changes into the uploaded branch.

How to host on Heroku

For your dynamic website

  1. First of all you'll need to download heroku, on mac this is done by using homebrew in the terminal as shown below. On Windows, click the installer button. For Ubuntu users, write the command shown below.
  2. Then you'll need to write: heroku login. This will open your browser and log in automatically. If everything went well you should see this in your terminal.
  3. Work your way in to your repository and create a file named "Procfile", the file naming is very strict.
  4. Open Procfile in a text editor program and write exactly this inside the file.
  5. Head over to your terminal again and write "heroku create". You will then be given a randomly generated hosting name and a link.
  6. To push your branch to your new site write: "git push heroku [ branch name ] ". If it doesnt work it is most likely that you have some errors with your heroku configuration or procfile.

    7. That's it! Your heroku site will automatically update every time you push to github, just like github pages.

Conclusion

    GitHub is for those who want

  • Quicker hosting
  • Easier hosting
  • Unlimited static hosting

    Heroku is for those who want

  • More copmlex hosting
  • More proffesional hosting
  • Limited dynamic hosting

Heroku and Github are both very powerful tools that can be used for online hosting, however you should chose your alternative based on what you want to upload. If you just want a little project uploaded as a folder without using express.js Github is the best choice. If you want a more server-like website that uses requests, responses and hidden files you should pick Heroku.
